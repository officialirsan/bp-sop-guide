SEVERITY

Severity Reference — P0 hingga P3

P0 Ack ≤ 1 jam P1 Ack ≤ 2 jam P2 Ack ≤ 4 jam P3 Ack ≤ 1 hari

Referensi cepat untuk semua anggota tim dalam menentukan dan menangani severity tiket klien. SLA dihitung dalam jam kerja (09.00–17.00 WIB, Senin–Jumat) kecuali kontrak hotline 24/7.

Severity Definisi Contoh First Response Workaround Resolution Postmortem
P0 Critical Layanan utama tidak bisa diakses / data loss aktif Sistem down total, tidak bisa login, data terhapus ≤ 1 jam ≤ 2 jam ≤ 4 jam Wajib
P1 High Fitur utama tidak berfungsi, ada dampak bisnis signifikan Laporan tidak bisa diexport, notifikasi tidak terkirim, API error 500 ≤ 2 jam ≤ 4 jam ≤ 1 hari kerja Wajib
P2 Medium Fitur non-kritis terdampak, ada workaround memadai Filter tidak bekerja, tampilan salah di browser tertentu ≤ 4 jam ≤ 1 hari kerja ≤ 3 hari kerja Opsional
P3 Low Minor issue, estetika, pertanyaan, enhancement minor Typo label, warna salah, pertanyaan cara pakai fitur ≤ 1 hari kerja N/A ≤ 5 hari kerja Tidak
